## Known, expected characteristics (not defects)

- Both goldens render CJK/Devanagari script runs as "tofu" replacement-glyph
  boxes. This is a property of the EXISTING, UNCHANGED `shux-raster` bundled
  font chain (JetBrains Mono NF + Noto Math/Symbols/Symbols2/Emoji — no CJK
  or Devanagari font in the bundle), not something introduced by
  `pane.glance`. LENS-R-013 requires reusing shux-raster unchanged, and this
  agent did not touch its font logic. The text side of the golden (the
  `.txt` files) DOES carry the correct Unicode codepoints — only the PIXEL
  rendering falls back to tofu for scripts the bundled fonts don't cover.
